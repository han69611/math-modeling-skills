# 赛题分析与建模设计

本 skill 承接 `1start-mathmodel`，目标是将赛题读薄：拆解子问题、建立变量与符号体系、写出模型公式与约束、确定求解策略，并输出建模设计报告。

## 必须产出

- `docs/REQUIREMENTS.md`：题目信息完整记录
- `reports/ANALYSIS_MODELING_REPORT.md`：建模设计报告
- `docs/ASSUMPTIONS.md`：假设清单（由 `03-hypothesis` 更新）

## 阶段边界

- 本阶段结束时，每个子问题必须已定义好目标函数、约束、变量表和求解算法。
- 本阶段不写代码，不跑实验。
- 本阶段输出全部交由 `04-coding` 读取。

## 审批节点（必须执行）

本阶段完成后，必须：
1. 向用户呈现建模报告的主要内容。
2. 通过 `present_files` 展示报告文件。
3. 明确询问用户：是否同意上述建模方案？是否需要调整？
4. **等待用户确认后再标记本阶段完成，进入编码阶段。**
