# 05-result-analysis — 结果分析与迭代决策

本 skill 承接 `04-coding-visual`。代码跑完后，先分析结果、解释原因、检查异常、判断是否需要重新建模。

## 核心原则

- 结果分析不是“报数字”，而是“讲故事”
- 每个数字都要回答：为什么是这个值？合理吗？说明什么？
- 异常结果不是错误，是发现——但必须解释原因

## 必须产出

- `reports/RESULTS_ANALYSIS.md`：完整的结果分析报告

## 分析框架：五层递进

### 第一层：结果报告（是什么）
### 第二层：原因解释（为什么）
### 第三层：异常检测（有没有问题）
### 第四层：结果翻译（意味着什么）
### 第五层：迭代决策（要不要重新建模）
