# 07-result-analysis — 结果分析与基准对比

本 skill 承接 `06-experiment-manager`。目标是分析结果、与Baseline对比、统计检验、判断是否需要迭代。

## 核心原则

- 结果分析不是“报数字”，而是“讲故事”
- 不只分析自己的模型，必须与Baseline对比
- 差异必须有统计检验支持

## 必须产出

- `reports/RESULTS_ANALYSIS.md`：完整的结果分析报告

## 分析框架：六层递进

### 第一层：结果报告（是什么）
### 第二层：Baseline对比（比别人好多少）
### 第三层：统计检验（差异是否显著）
### 第四层：原因解释（为什么）
### 第五层：异常检测
### 第六层：迭代决策
